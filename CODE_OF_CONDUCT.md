# Code of Conduct

HELM adopts the Contributor Covenant Code of Conduct, version 2.1:
<https://www.contributor-covenant.org/version/2/1/code_of_conduct/>.

## Expected Behavior

Contributors are expected to be respectful, constructive, and precise. Healthy
technical disagreement is welcome when it stays focused on evidence, tradeoffs,
and the project scope in `README.md` and `docs/KERNEL_SCOPE.md`.

## Unacceptable Behavior

Harassment, abuse, discriminatory language, sustained disruption, private
information disclosure, and bad-faith security handling are not acceptable in
project spaces.

## Scope

This code of conduct applies in repository issues, pull requests, discussions,
project communication channels, events, and any public interaction where someone
is representing HELM.

## Enforcement

Reports go to `conduct@mindburn.org`. Maintainers handle reports under
`GOVERNANCE.md`; anyone directly involved in a report must recuse themselves.
Security-sensitive material should follow `SECURITY.md` instead of this public
process.

Maintainers may remove comments, close threads, restrict participation, or take
other proportionate action to protect the project and its contributors.
