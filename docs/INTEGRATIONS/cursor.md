# Cursor Integration

Use Cursor with HELM through MCP configuration generated by the HELM CLI.

## Print MCP Config

```bash
helm-ai-kernel mcp print-config --client cursor
```

Install the printed config in Cursor according to the client location shown by
the command output.

## Run The Boundary

```bash
helm-ai-kernel serve --policy ./release.high_risk.v3.toml
```

## Verify Receipts

```bash
helm-ai-kernel receipts tail --agent <agent-id> --server http://127.0.0.1:7714
```

## Debug

| Symptom | First check |
| --- | --- |
| Cursor does not list the HELM MCP server | Re-run `helm-ai-kernel mcp print-config --client cursor` and compare the installed config. |
| Calls are denied | Check the server ID, tool name, schema pin, and policy fixture. |
| Receipts are missing | Confirm the boundary is running and tail receipts for the correct agent ID. |
