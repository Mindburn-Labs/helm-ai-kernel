# AI Security Categories

AI security tools do not all sit in the same layer. The safe comparison is a
boundary comparison, not a winner-take-all claim.

| Layer | Example public category | What it usually checks | Where HELM fits |
| --- | --- | --- | --- |
| Message-policy enforcement | ZeroDrift-style communication controls | AI-written messages, review rules, delivery gates | Adjacent layer |
| Security testing | Strix-style AI security testing loops | vulnerabilities, test runs, exploit paths, reports | Adjacent layer |
| Posture and monitoring | scanner or dashboard tools | inventory, risk signals, logs, alerts | Adjacent layer |
| Execution firewall | HELM AI Kernel | proposed side effects before dispatch | Core layer |

## HELM's Narrow Claim

HELM is a fail-closed execution firewall for routed agent actions. Models
propose. HELM decides. The public proof is a verdict, receipt, and verifiable
EvidencePack material.

## Comparison Rules

- Use public evidence only.
- Say "different layer" before saying "better".
- Name the routed action and proof artifact for any HELM claim.
- Do not imply HELM governs an action that never crossed the boundary.
