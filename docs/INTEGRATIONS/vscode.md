# VS Code Integration

Use VS Code with HELM through MCP configuration generated by the HELM CLI.

## Print MCP Config

```bash
helm-ai-kernel mcp print-config --client vscode
```

Install the printed config in the VS Code MCP-capable surface you use.

## Run The Boundary

```bash
helm-ai-kernel serve --policy ./release.high_risk.v3.toml
```

## Verify Receipts

```bash
helm-ai-kernel receipts tail --agent <agent-id> --server http://127.0.0.1:7714
```

## Debug

| Symptom | First check |
| --- | --- |
| VS Code does not list the HELM MCP server | Re-run `helm-ai-kernel mcp print-config --client vscode` and compare the installed config. |
| Tool call is denied | Inspect the policy, tool schema pin, and latest receipt. |
| No receipt appears | Confirm the app is calling the HELM MCP server, not the upstream tool server directly. |
