# Support

Use the lowest-sensitivity channel that fits the request.

| Need | Channel |
| --- | --- |
| Bug report with reproduction steps | GitHub issue using the bug template |
| Feature proposal or public integration question | GitHub discussion |
| Contribution workflow question | GitHub discussion or `CONTRIBUTING.md` |
| Security vulnerability | `security@mindburn.org`; do not open a public issue |
| Conduct report | `conduct@mindburn.org` |
| Maintainer contact | `maintainers@mindburn.org` |

Include the HELM version, command used, policy bundle hash when relevant,
receipt ID or evidence bundle path when relevant, and verifier output. Do not
include provider keys, private prompts, customer data, or unredacted production
receipts.
