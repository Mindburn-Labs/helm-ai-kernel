# Windsurf Integration

Use Windsurf with HELM through MCP configuration generated by the HELM CLI.

## Print MCP Config

```bash
helm-ai-kernel mcp print-config --client windsurf
```

Install the printed config in Windsurf according to the client location shown by
the command output.

## Run The Boundary

```bash
helm-ai-kernel serve --policy ./release.high_risk.v3.toml
```

## Verify Receipts

```bash
helm-ai-kernel receipts tail --agent <agent-id> --server http://127.0.0.1:7714
```

## Debug

| Symptom | First check |
| --- | --- |
| Windsurf does not list the HELM MCP server | Re-run `helm-ai-kernel mcp print-config --client windsurf` and compare the installed config. |
| Tool is quarantined | Check discovery metadata, approval state, and pinned schema hash. |
| Receipts are missing | Confirm the boundary is running and the client points at HELM. |
